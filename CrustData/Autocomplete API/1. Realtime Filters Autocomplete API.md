# Realtime Filters Autocomplete API

### [ 🚀 Try Now ](/api#tag/auxiliary-api/GET/screener/linkedin_filter/autocomplete)

This endpoint provides autocomplete suggestions for filters, helping users find relevant filter values while building their search queries. You can use these 

## Available Filters
- `region` - Geographic regions and locations
- `industry` - Industry categories
- `title` - Job titles
- `school` - Educational institutions and schools 

## Endpoint

```
GET /screener/linkedin_filter/autocomplete
```

## Request Parameters

| Parameter     | Type    | Description                                                | Required | Default |
|--------------|---------|------------------------------------------------------------| -------- |---------|
| `filter_type`| string  | Type of filter to get suggestions for.<br/>Possible values:<br/>• `region` - Geographic regions and locations<br/>• `industry` - Industry categories<br/>• `title` - Job titles<br/>• `school` - Educational institutions and schools<br/>  | Yes | - |
| `query`      | string  | Search text to get autocomplete suggestions. Example: "san" for "San Francisco" | Yes | - |
| `count`      | integer | Number of suggestions to return (max: 50) | No | 10 |
| `start`      | integer | Starting index for pagination. Use with `count` to paginate through results | No | 0 |

## Filter Types

The `filter_type` parameter accepts the following values:

- `region` - Geographic regions and locations
- `industry` - Industry categories
- `title` - Job titles
- `school` - Educational institutions and schools 

## Credit Usage

- This endpoint does not consume any credits
